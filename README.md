# YKI Tracker — мультиязычный трекер подготовки к YKI

Личные трекеры подготовки к **YKI** (keskitaso) для нескольких студентов и языков.
Фокус на **kirjoittaminen** (письмо) и **puhuminen** (говорение). Цель каждого — надёжная **3**
(минимум усилий). Без бэкенда, работает офлайн: markdown-память + статический HTML-дашборд.

## Языковые дорожки
| Папка | Студент | Целевой язык | Статус |
|---|---|---|---|
| `lang/svenska/` | Даниэль | шведский | в работе (5 сессий) |
| `lang/suomi/` | — | финский | стартовый скелет |

## Структура репо
```
yki-tracker/
├── AGENTS.md                 ← роутер: в начале сессии определить язык
├── core/                     ← ОБЩИЙ движок, правится один раз
│   ├── AGENTS.md               протокол сессий (язык-нейтральный)
│   └── dashboard/
│       ├── engine.js            рендер дашборда (весь текст из data.js)
│       └── styles.css           стили
└── lang/
    ├── svenska/               ← Даниэль: полный контент + прогресс
    │   ├── config progress sessions reviews bank curriculum
    │   └── dashboard/{index.html, data.js}
    └── suomi/                 ← финский: скелет (TODO-плейсхолдеры)
        ├── config progress sessions reviews bank curriculum
        └── dashboard/{index.html, data.js}
```

**Почему так:** общий движок (протокол сессий + вёрстка дашборда) правится **один раз** в `core/`
и применяется ко всем языкам. Каждый язык живёт в своей папке и не зависит от остальных —
поэтому правки общих правил/вёрстки не создают перекрёстных merge-конфликтов между языками.

## Как заниматься (любым языком)
1. Сказать ассистенту «занимаемся» + язык (напр. «сессия svenska» / «сессия suomi»).
2. Ассистент читает `core/AGENTS.md` (протокол) + память студента в `lang/<язык>/progress/`,
   даёт промпт в стиле YKI.
3. Студент отвечает на целевом языке → исправления + правило под каждую ошибку.
4. В конце — оценка и обновление памяти + `lang/<язык>/dashboard/data.js`.
5. Открыть `lang/<язык>/dashboard/index.html` (или F5) — увидеть прогресс.

## Дашборд
Каждая дорожка открывает свой `lang/<язык>/dashboard/index.html`. Тонкая обёртка подключает
свой `data.js` (контент + весь видимый текст) и общий движок из `core/dashboard/engine.js`.
Без бэкенда, работает офлайн на `file://`.
