# AGENTS.md — мультиязычный трекер подготовки к YKI

Этот файл — инструкция для ассистента (меня). **Читать в начале КАЖДОЙ сессии.**

Это **мультиязычный проект**: несколько студентов готовятся к YKI (keskitaso),
каждый к своему языку. Каждый язык живёт в своей папке и **не зависит** от других.

## Языковые дорожки
| Папка | Студент | Целевой язык | Фокус |
|---|---|---|---|
| `lang/svenska/` | Даниэль | шведский | kirjoittaminen (письмо) + puhuminen (говорение) |
| `lang/suomi/` | — | финский | kirjoittaminen (письмо) + puhuminen (говорение) |

## В начале сессии — определить язык ⭐
1. Если студент прямо не сказал язык — спросить (или вывести из контекста/последней сессии).
2. Дальше работать **только внутри папки `lang/<язык>/`** — это «проект» этого студента.
3. **Прочитать операционный протокол:** `core/AGENTS.md` (язык-нейтральный каркас —
   лестница сложности, ворота мастерства, цикл сессии, spaced repetition, обновление файлов).
4. **Прочитать память студента** внутри `lang/<язык>/`:
   - `progress/profile.md`, `progress/mastery.md`, `reviews/due.md`, `sessions/log.md`.

## Архитектура (что где править, чтобы не было конфликтов)
- `core/AGENTS.md` — **общий протокол сессий**. Правится один раз, применяется ко всем языкам.
- `core/dashboard/` — **общий движок дашборда** (`engine.js`, `styles.css`). Правится один раз.
- `lang/<язык>/` — **всё язык-специфичное**: студент, прогресс, сессии, curriculum, банк заданий,
  и `dashboard/data.js` + тонкая обёртка `dashboard/index.html`.
- Правка общего правила/вёрстки → трогаем только `core/`. Каждый язык получает её автоматически,
  перекрёстных merge-конфликтов нет.

> Подробный протокол — в `core/AGENTS.md`. Подробный обзор проекта — в `README.md`.
