# YKI Svenska — дорожка подготовки (шведский)

Личный трекер подготовки к **YKI Swedish** (keskitaso), фокус на **письме** (kirjoittaminen)
и **говорении** (puhuminen). Цель: надёжная **3** с минимумом усилий.

> Это языковая дорожка внутри общего мультиязычного трекера. Общий протокол сессий и
> движок дашборда живут в `core/` (в корне репо). Здесь — только шведский контент.

## Что внутри (`lang/svenska/`)
```
lang/svenska/
├── config/student.md          параметры студента
├── PLAN.md                    фазы и таймлайн
├── curriculum/                опорные материалы (cognate-теги ↔EN/↔RU)
│   ├── themes.md              10 тем YKI + кластеры слов
│   ├── functions.md           12 коммуникативных функций + каркасы
│   ├── writing-templates.md   каркасы писем (формал./неформал.)
│   ├── speaking-toolkit.md    маркеры, филлеры, фразы взаимодействия
│   └── grammar-essentials.md  грамматика high-yield («EN с твистом»)
├── bank/                      банк заданий в стиле YKI
│   ├── writing-prompts.md
│   └── speaking-prompts.md
├── progress/                  ПАМЯТЬ о студенте
│   ├── profile.md             текущий снимок уровня
│   ├── mastery.md             матрица тема×функция + даты SR
│   ├── mistakes.md            лог ошибок по паттернам
│   └── assessment.md          скользящие оценки 1–6
├── sessions/                  история сессий
│   └── log.md
├── reviews/due.md             очередь повторения на сегодня
└── dashboard/
    ├── index.html             открыть в браузере
    └── data.js                данные (обновляются после сессии)
```

## Как заниматься
1. Сказать ассистенту «занимаемся шведским» / «сессия svenska».
2. Ассистент читает `core/AGENTS.md` (протокол) + память здесь (`progress/`), даёт промпт в стиле YKI.
3. Отвечаешь по-шведски → исправления + правило.
4. В конце — оценка и обновление всех файлов + `dashboard/data.js`.
5. Открыть `dashboard/index.html` (или F5) — увидеть прогресс.

## Дашборд
Открыть `dashboard/index.html` в браузере. Данные лежат в `dashboard/data.js` —
обновляются после каждой сессии. Вёрстку/логику рендера движок берёт из `core/dashboard/`.
Без бэкенда, работает офлайн.
